# Knowledge Graph

Each hive contains a `KnowledgeGraph` ŌĆö a `networkx.MultiDiGraph` (directed multigraph) stored as JSON.

The knowledge graph represents interconnected concepts and papers using three node types and multiple relationship types.

## Node types

### Paper

A research paper from arXiv, including title, authors, abstract, publication date, and classification categories.

```json
{
  "id": "paper:1706.03762",
  "type": "paper",
  "label": "Attention Is All You Need",
  "arxiv_id": "1706.03762",
  "authors": "Vaswani et al.",
  "abstract": "The dominant sequence transduction models are based on complex recurrent or convolutional neural networks...",
  "published": "2017-06-12",
  "categories": ["cs.CL", "cs.LG"]
}
```

### Concept

A keyword, term, idea, or method extracted from papers (e.g., "Self-Attention", "Transformer Architecture"). Concept nodes contain a definition and can be used across multiple papers.

```json
{
  "id": "concept:Self-Attention",
  "type": "concept",
  "label": "Self-Attention",
  "definition": "An attention mechanism that computes attention scores between all pairs of positions in a sequence.",
  "concept_type": "algorithmic",
  "graph_id": "hive-name"
}
```

**Common concept types:**
- `algorithmic`: Core algorithms and techniques
- `architecture`: System designs and structures
- `method`: Specific approaches and procedures
- `phenomenon`: Observed behaviors and effects
- `entity`: Named objects and concepts

**Relationships:** Concepts appear in papers via `related_to` edges, and papers introduce concepts via `introduces` edges.

### Graph Ref

A reference to another hive for cross-domain linking (e.g., `llms` points to the llms hive).

```json
{
  "id": "ref:llms",
  "type": "graph_ref",
  "label": "llms",
  "target_graph_id": "llms",
  "relation": "references"
}
```

## Edge types

Edges represent relationships between nodes. Each edge has a directed meaning:

| Relation | Direction | Meaning |
|---|---|---|
| `related_to` | concept Ōåö paper | Concept is discussed in the paper |
| `introduces` | paper ŌåÆ concept | Paper introduces the concept |
| `references` | graph_ref ŌåÆ hive | Hive references another hive |
| `extends` | cross-hive | Concept extends a concept in another hive |
| `uses` | cross-hive | Concept uses a concept in another hive |
| `contradicts` | cross-hive | Concept contradicts a concept in another hive |

Cross-graph edges have `cross_graph: true` and `target_graph` set to the target hive name.

## Persistence

Graphs are serialized using `networkx.node_link_data()` and written as indented JSON. On every `save()`, the previous state is copied to a timestamped backup file.

## Ollama Definition Enrichment

Concept nodes can optionally be enriched with short definitions generated by a local [Ollama](https://ollama.ai) model (default: `gemma4:31b-mlx`). These definitions are stored in the `definition` field and displayed as tooltips in the web UI.

**CLI:** `python -m hivemind ingest "Graph Neural Network" --ollama`  
**API:** `POST /api/enrich-definitions` with `{"hive": "hive-name"}`  
**Dashboard:** Select a hive and click the **Enrich** button.

When enabled, definitions are auto-generated during concept ingestion or can be batch-generated for existing concepts via the **Enrich** button. Configurable via environment variables:

| Variable | Default | Description |
|---|---|---|
| `USE_OLLAMA_DEFINITIONS` | `false` | Enable (or use `--ollama` flag) |
| `OLLAMA_URL` | `http://localhost:11434` | Ollama server URL |
| `OLLAMA_MODEL` | `gemma4:31b-mlx` | Model name |
| `OLLAMA_TIMEOUT` | `30` | Request timeout (seconds) |
