Features

What makes HiveMind different

Traditional knowledge graphs are isolated. HiveMind weaves them into a living federation where each graph is a hive in a larger mind.

◇

Multiple KGs

Separate knowledge graphs per topic, domain, or project — each with its own nodes, edges, and storage.

↩

Graph Nesting

Embed a reference to an entire knowledge graph as a single node. Drill into any hive with one click.

⇄

Cross-Graph Edges

Connect concepts across hives. attention in your Transformers graph can link to message passing in GNNs.

◎

Meta-Graph

An automatic top-level graph shows how your hives connect. See the forest, not just the trees.

☌

Unified Search

Search across all hives at once. Find related concepts scattered across different knowledge domains.

☷

Live Dashboard

D3.js-powered force-directed graph with zoom, pan, and per-hive drill-down. No build step required.

Architecture

How HiveMind works

Each hive is a standalone knowledge_graph.json file. A federation layer loads them all, tracks cross-graph edges, and builds a live meta-graph.

CLI Reference

Command-line interface

HiveMind ships with a rich CLI. All commands work offline — no database required.

$ hivemind init my-hive → create a new knowledge graph
$ hivemind link a b → nest hive b inside hive a
$ hivemind connect a concept:x b concept:y → cross-graph edge
$ hivemind search "attention" → unified search across all hives
$ hivemind import /path/to/arxiv-to-obsidian → import existing KG
$ hivemind serve → launch dashboard at :9090
$ hivemind stats → federation statistics 
# Create two hives and link them
$ hivemind init transformers
Created hive at hives/transformers
$ hivemind init gnns
Created hive at hives/gnns
$ hivemind link transformers gnns
Linked 'transformers' -> 'gnns' (references)

# Connect concepts across hives
$ hivemind connect transformers attention gnns "message passing"
Connected transformers:attention -related_to-> gnns:message passing

# Query everything
$ hivemind search attention
transformers  concept:attention        concept  attention
$ hivemind stats

  Concepts:         6

Ecosystem

Works with arxiv-to-obsidian

Import any arxiv-to-obsidian knowledge graph directly. Every paper, concept, and relation becomes a hive in your federation.

arxiv-to-obsidian
A self-organising research knowledge base that fetches arXiv papers, extracts concepts via LLM, builds a knowledge graph, and exports to Obsidian markdown.
→
HiveMind Federation
One hivemind import wraps an entire arxiv-to-obsidian graph as a hive, ready to link with other hives. 
# Import an existing arxiv-to-obsidian project
$ hivemind import ~/projects/arxiv-to-obsidian --name nlp-papers
Imported as hive 'nlp-papers'

$ hivemind link nlp-papers transformers
Linked 'nlp-papers' -> 'transformers' (references)
